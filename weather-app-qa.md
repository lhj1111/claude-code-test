# 날씨 앱 QA 테스트 시나리오

## 1. E2E 테스트 시나리오

### E2E-01: 도시 검색 및 날씨 조회
- **전제조건**: 앱이 정상 로딩된 상태
- **단계**:
  1. 검색 입력란에 "서울"을 입력한다
  2. 검색 결과 드롭다운에서 "서울, 대한민국"을 선택한다
  3. 날씨 상세 페이지로 이동한다
- **기대결과**:
  - 현재 온도, 습도, 풍속, 날씨 아이콘이 표시된다
  - 5일 예보가 정상 렌더링된다
  - URL에 도시 식별자가 포함된다

### E2E-02: 즐겨찾기 추가
- **전제조건**: "서울" 날씨 상세 페이지에 있는 상태
- **단계**:
  1. 즐겨찾기 추가 버튼(빈 별 아이콘)을 클릭한다
  2. 홈 페이지로 돌아간다
- **기대결과**:
  - 별 아이콘이 채워진 상태로 변경된다
  - 홈 페이지 즐겨찾기 목록에 "서울"이 표시된다
  - 페이지 새로고침 후에도 즐겨찾기가 유지된다

### E2E-03: 즐겨찾기 삭제
- **전제조건**: "서울"이 즐겨찾기에 등록된 상태
- **단계**:
  1. 홈 페이지 즐겨찾기 목록에서 "서울"의 삭제 버튼을 클릭한다
  2. 확인 다이얼로그에서 "삭제"를 선택한다
- **기대결과**:
  - 즐겨찾기 목록에서 "서울"이 제거된다
  - 날씨 상세 페이지의 별 아이콘이 빈 상태로 돌아간다

### E2E-04: 알림 설정
- **전제조건**: 로그인 상태, 즐겨찾기에 도시가 1개 이상 등록됨
- **단계**:
  1. 설정 페이지로 이동한다
  2. 알림 토글을 활성화한다
  3. 알림 시간을 "오전 7:00"으로 설정한다
  4. 알림 대상 도시로 "서울"을 선택한다
  5. 저장 버튼을 클릭한다
- **기대결과**:
  - "알림이 설정되었습니다" 토스트 메시지가 표시된다
  - 설정 페이지 재방문 시 저장된 값이 유지된다

### E2E-05: 검색 자동완성 및 최근 검색
- **전제조건**: 앱이 정상 로딩된 상태
- **단계**:
  1. 검색 입력란에 "뉴"를 입력한다
  2. 자동완성 목록에서 "뉴욕, 미국"을 선택한다
  3. 날씨를 확인 후 홈으로 돌아간다
  4. 다시 검색 입력란을 포커스한다
- **기대결과**:
  - 자동완성 목록에 "뉴욕", "뉴델리" 등 관련 도시가 표시된다
  - 최근 검색 섹션에 "뉴욕"이 표시된다

### E2E-06: 다국어 도시 검색
- **전제조건**: 앱이 정상 로딩된 상태
- **단계**:
  1. 검색 입력란에 "Tokyo"를 입력한다
  2. 검색 결과에서 "Tokyo, Japan"을 선택한다
- **기대결과**:
  - 영문 도시명으로도 정상 검색된다
  - 날씨 데이터가 올바르게 표시된다

## 2. API 테스트 케이스

### 2.1 정상 케이스

| ID | 엔드포인트 | 메서드 | 입력 | 기대 상태 코드 | 기대 응답 |
|----|-----------|--------|------|---------------|----------|
| API-01 | `/api/weather?city=Seoul` | GET | city=Seoul | 200 | `{ data: { city, temperature, humidity, windSpeed, ... } }` |
| API-02 | `/api/weather/forecast?city=Seoul&days=5` | GET | city, days | 200 | `{ data: { forecasts: [...] } }` (5개 항목) |
| API-03 | `/api/favorites` | POST | `{ cityId: "seoul" }` | 201 | `{ data: { id, cityId, createdAt } }` |
| API-04 | `/api/favorites` | GET | - | 200 | `{ data: [{ id, cityId, ... }] }` |
| API-05 | `/api/favorites/:id` | DELETE | id (uuid) | 204 | 빈 응답 |
| API-06 | `/api/notifications` | PATCH | `{ enabled: true, time: "07:00" }` | 200 | `{ data: { enabled, time, cities } }` |
| API-07 | `/api/cities/search?q=서울` | GET | q=서울 | 200 | `{ data: [{ id, name, country, lat, lon }] }` |

### 2.2 오류 케이스

| ID | 엔드포인트 | 메서드 | 입력 | 기대 상태 코드 | 기대 응답 |
|----|-----------|--------|------|---------------|----------|
| API-E01 | `/api/weather?city=` | GET | 빈 city | 422 | `{ error: { message: "city is required", code: "VALIDATION_ERROR" } }` |
| API-E02 | `/api/weather?city=zzzznotacity` | GET | 존재하지 않는 도시 | 404 | `{ error: { message: "City not found", code: "NOT_FOUND" } }` |
| API-E03 | `/api/favorites` | POST | `{ cityId: "" }` | 422 | `{ error: { message: "cityId is required", code: "VALIDATION_ERROR" } }` |
| API-E04 | `/api/favorites` | POST | 중복 cityId | 422 | `{ error: { message: "Already in favorites", code: "DUPLICATE" } }` |
| API-E05 | `/api/favorites/invalid-uuid` | DELETE | 잘못된 uuid | 422 | `{ error: { message: "Invalid id format", code: "VALIDATION_ERROR" } }` |
| API-E06 | `/api/favorites/00000000-0000-0000-0000-000000000000` | DELETE | 없는 id | 404 | `{ error: { message: "Favorite not found", code: "NOT_FOUND" } }` |
| API-E07 | `/api/notifications` | PATCH | `{ time: "25:00" }` | 422 | `{ error: { message: "Invalid time format", code: "VALIDATION_ERROR" } }` |

## 3. 엣지 케이스

### 네트워크 및 서버 오류

| ID | 시나리오 | 테스트 방법 | 기대 동작 |
|----|---------|------------|----------|
| EDGE-01 | 네트워크 연결 끊김 | DevTools에서 네트워크 오프라인 전환 | "네트워크 연결을 확인해주세요" 오류 메시지 표시, 재시도 버튼 제공 |
| EDGE-02 | API 서버 다운 (5xx) | API 서버 중단 상태에서 요청 | "일시적인 오류가 발생했습니다" 메시지 표시, 자동 재시도 (TanStack Query retry) |
| EDGE-03 | 응답 타임아웃 | API 응답을 10초 이상 지연 | 로딩 스피너 표시 후 타임아웃 오류 메시지, 재시도 옵션 제공 |

### 입력 및 데이터 엣지 케이스

| ID | 시나리오 | 테스트 방법 | 기대 동작 |
|----|---------|------------|----------|
| EDGE-04 | 특수문자 도시 검색 | `<script>alert(1)</script>` 입력 | XSS 방지, 입력값 이스케이프 처리, 검색 결과 없음 표시 |
| EDGE-05 | 매우 긴 도시명 입력 | 500자 이상 문자열 입력 | 입력 길이 제한 또는 유효성 오류 메시지 |
| EDGE-06 | 빈 검색어 제출 | 아무것도 입력하지 않고 검색 | 검색 실행되지 않거나 안내 메시지 표시 |
| EDGE-07 | 날씨 데이터 일부 누락 | API 응답에서 일부 필드 null | "데이터 없음" 또는 "-" 표시, 앱 크래시 없음 |
| EDGE-08 | 즐겨찾기 최대 개수 초과 | 최대치(예: 20개) 이상 추가 시도 | "즐겨찾기는 최대 20개까지 추가할 수 있습니다" 메시지 |

### 동시성 및 상태 엣지 케이스

| ID | 시나리오 | 테스트 방법 | 기대 동작 |
|----|---------|------------|----------|
| EDGE-09 | 즐겨찾기 빠른 중복 클릭 | 추가 버튼을 연속 3회 빠르게 클릭 | 1회만 등록, 중복 요청 방지 (debounce 또는 optimistic lock) |
| EDGE-10 | 탭 간 상태 동기화 | 두 탭에서 앱 열고, 한 쪽에서 즐겨찾기 변경 | 다른 탭에서 포커스 시 최신 상태 반영 (또는 새로고침 시 반영) |
| EDGE-11 | 검색 중 빠른 입력 변경 | "서" → "서울" → "서울시" 빠르게 입력 | 마지막 입력에 대한 결과만 표시 (race condition 방지) |
