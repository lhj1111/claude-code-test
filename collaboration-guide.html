Beginner Guide

Codex와 Claude Code를
같이 쓰는 가장 안전한 방법

두 AI를 동시에 쓸 수는 있지만, 서로 마음대로 대화하게 두는 방식은 초보자에게 불리합니다. 가장 추천하는 방식은 역할을 나누고, 작업 내용을 파일로 넘기고, 마지막 판단은 사람이 하는 구조입니다.

권장 방식: 역할 분리 작업 전달: HANDOFF.md 최종 승인: 사용자
한 줄 요약 Claude는 분석과 리뷰, Codex는 구현과 테스트를 맡기는 방식이 가장 안정적입니다.
왜 좋은가 충돌이 적고, 누가 무엇을 했는지 추적하기 쉽고, 초보자도 흐름을 이해하기 쉽습니다.
주의할 점 같은 파일을 두 AI가 동시에 수정하게 두면 결과가 쉽게 꼬입니다.

전체 구동 방식 시각자료

이 그림만 봐도 누가 무엇을 하고, 정보가 어디를 통해 이동하는지 이해할 수 있게 구성했습니다.

2. 실제 작업 순서
1

요청

사용자가 해결할 문제나 만들 기능을 설명합니다.

2

분석

Claude Code가 구조를 읽고 원인, 접근 방식, 주의점을 정리합니다.

3

전달

핵심 내용을 HANDOFF.md 같은 공용 문서에 적어 다음 작업자에게 넘깁니다.

4

구현

Codex가 문서를 기준으로 코드 수정과 테스트를 수행합니다.

5

검토

Claude와 사용자가 결과를 확인한 뒤 최종 반영 여부를 결정합니다.

핵심 흐름은 "사용자 지시 → Claude 분석 → 문서로 전달 → Codex 구현 → 사용자 승인"입니다.
핵심 개념

추천하지 않는 방식

두 AI가 직접 대화하면서 동시에 프로젝트를 수정하는 방식입니다.

  • 같은 파일을 동시에 바꾸면 충돌이 발생하기 쉽습니다.
  • 잘못된 가정이 서로 오가며 더 커질 수 있습니다.
  • 누가 최종 결정을 내렸는지 흐려집니다.

추천하는 방식

각 AI의 역할을 분리하고, 작업 내용을 파일로 넘겨주면서 협업하는 방식입니다.

  • 역할이 명확해서 초보자도 흐름을 따라가기 쉽습니다.
  • 중간 기록이 남아 다음 작업을 이어가기 쉽습니다.
  • 문제가 생겨도 어디서 꼬였는지 찾기 쉽습니다.
역할 나누기

가장 실용적인 역할 분담

두 AI를 잘 쓰려면 서로 같은 일을 시키지 말고, 잘하는 영역을 분리하는 편이 좋습니다.

Claude Code

  • 프로젝트 구조 파악
  • 문제 원인 분석
  • 구현 계획 작성
  • 코드 리뷰와 리스크 점검

Codex

  • 코드 수정
  • 테스트 실행
  • 에러 수정
  • 최종 변경사항 정리
초보자라면 "생각하는 AI 1명 + 실제 수정하는 AI 1명"으로 이해하면 됩니다.
실제 작업 순서

이 순서대로 쓰면 됩니다

1
Claude에게 먼저 분석을 맡깁니다 예: "이 버그 원인 분석하고 수정 계획을 정리해줘"
2
분석 결과를 기록합니다 `HANDOFF.md` 같은 파일에 현재 문제, 원인, 해야 할 작업을 적습니다.
3
Codex에게 구현을 맡깁니다 예: "`HANDOFF.md` 기준으로 코드 수정하고 테스트해줘"
4
Claude에게 리뷰를 맡깁니다 Codex가 만든 변경사항이 맞는지, 부작용은 없는지 확인합니다.
5
마지막 승인은 사람이 합니다 머지하거나 반영하기 전에 반드시 직접 확인합니다.
공용 파일 예시

AGENTS.md

각 AI가 무슨 역할을 맡는지 적는 파일입니다.

# AGENTS.md
Claude Code:
- architecture analysis
- code review
- task planning

Codex:
- implementation
- test execution
- bug fixing

Rules:
- Never edit the same file concurrently
- Write task context to HANDOFF.md

HANDOFF.md

현재 상태와 다음 작업을 넘기는 파일입니다.

# HANDOFF.md
Current goal:
- Fix todo filter bug

Claude notes:
- Root cause seems to be in frontend filter state handling
- Add one regression test if possible

Next for Codex:
- Patch the bug
- Run related tests
- Summarize changes
초보자용 체크리스트

이 다섯 가지만 지키면 됩니다

  1. 한 번에 한 AI만 실제 코드 수정하게 하기
  2. 분석 결과를 파일에 남기기
  3. 중요 작업은 바로 구현하지 말고 먼저 계획 확인하기
  4. 테스트 결과와 변경 요약을 항상 받기
  5. 최종 결정은 직접 하기
핵심은 "AI 둘이 자유롭게 떠들게 두는 것"이 아니라 "사람이 규칙을 정하고 작업을 넘기는 것"입니다.

이 문서는 입문자가 Codex와 Claude Code를 함께 사용할 때 가장 안전한 협업 방식을 이해하도록 만든 설명 자료입니다.
